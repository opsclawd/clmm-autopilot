# Specs Directory

This folder contains milestone specs that are mandatory inputs for implementation.

## Rules

- One milestone per branch/PR.
- A milestone spec file is required before implementation starts.
- Update the active milestone spec when scope, dependencies, scripts, or acceptance criteria change.
- If implementation and spec diverge, spec must be updated in the same PR.

## Naming

Use: `specs/<milestone>-<slug>.spec.md`

Examples:
- `specs/m0-scaffold.spec.md`
- `specs/m1-foundations.spec.md`
